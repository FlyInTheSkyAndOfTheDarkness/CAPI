# Боевой запуск с реальным amoCRM

Чтобы платформа реально отправляла конверсии, нужен **публичный HTTPS-адрес**
(amoCRM не принимает `localhost` ни в OAuth, ни в вебхуках) и **ваши реальные
доступы**. Ниже два пути: быстрый тест с локальной машины и продакшн на сервере.

---

## Путь A — быстрый тест с localhost через ngrok

1. Установите ngrok и зарегистрируйтесь (бесплатно), получите authtoken:
   https://ngrok.com/download → `ngrok config add-authtoken <ваш_токен>`.
2. Поднимите туннель на порт API:
   ```bash
   ngrok http 3001
   ```
   Скопируйте выданный `https://<...>.ngrok-free.app`.
3. Пропишите его в `apps/api/.env`:
   ```
   PUBLIC_API_URL=https://<...>.ngrok-free.app
   WEB_ORIGIN=http://localhost:5173
   ```
   и перезапустите API (`npm run dev:api`). Теперь Redirect URI и вебхук-URL в
   кабинете будут с публичным адресом.

> ngrok на бесплатном тарифе меняет URL при каждом рестарте — после смены нужно
> заново обновить `PUBLIC_API_URL` и Redirect URI в интеграции amoCRM.

## Путь B — продакшн (рекомендуется)

Разверните Docker-стек на сервере/VPS с доменом и HTTPS (см. README → «Деплой»):
`PUBLIC_API_URL=https://ваш-домен` стабилен и не меняется. Перед nginx поставьте
TLS-терминатор (Caddy/Traefik/Cloudflare).

---

## Шаги подключения amoCRM (одинаковы для A и B)

1. Кабинет CAPI → «Подключения CRM» → «+ Добавить CRM» → **amoCRM** → метод
   **OAuth**. Скопируйте показанный **Redirect URI**.
2. В amoCRM: Настройки → Интеграции → «Создать интеграцию» (внешняя). Вставьте
   Redirect URI, дайте доступ к CRM. Сохраните — получите **ID интеграции**
   (Client ID) и **Секретный ключ** (Client Secret).
3. Вернитесь в мастер CAPI, введите адрес аккаунта (`https://ваш.amocrm.ru`),
   Client ID и Secret → «Создать и продолжить» → «Авторизовать в amoCRM».
   После авторизации вебхук на события лидов **подпишется автоматически**.
4. Диагностика в карточке подключения должна показать зелёные галочки по
   авторизации и подписке вебхука.

## Направление Meta (чтобы конверсии реально уходили)

1. Events Manager → ваш пиксель → скопируйте **Pixel ID**.
2. Настройки → Conversions API → «Сгенерировать токен доступа» → **access token**.
3. Кабинет CAPI → «Направления» → «+ Добавить направление» → **Meta** → введите
   Pixel ID и токен. Можно указать Test Event Code для проверки в «Тестовых
   событиях» Events Manager.
4. Кнопка «Отправить тестовое событие» в мастере — убедиться, что токен верный.

## Маппинг и проверка

1. «Маппинг событий» → выберите подключение, направление, воронку и этап (из
   выпадающих списков — они подтянутся из amoCRM), задайте событие (`Lead`,
   `Purchase`), при желании «Передавать сумму».
2. Переведите в amoCRM тестовую сделку/лид на этот этап.
3. В «Логах доставки» появится событие; в «Тестовых событиях» Events Manager
   Meta — принятая конверсия. На дашборде и в «Аналитике» — статистика.
